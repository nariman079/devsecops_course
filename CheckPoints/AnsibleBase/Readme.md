# 🎓 Лабораторная работа: Основы Ansible
**Тема:** Автоматизация управления серверами с использованием Docker и Ansible.

## 📋 Цель работы
1. Настроить изолированное окружение для разработки (Python venv).
2. Развернуть кластер из 3-х виртуальных серверов Ubuntu с помощью Docker Compose.
3. Освоить базовые модули Ansible: установка ПО, управление пользователями, работа с шаблонами.

---

## 🛠 Часть 1. Подготовка окружения

### Шаг 1. Установка Docker
Для работы лаборатории необходим **Docker Desktop**.
*   **Windows/macOS:** Скачайте и установите с [официального сайта](https://www.docker.com/products/docker-desktop/). После установки убедитесь, что приложение запущено (зеленый индикатор).
*   **Linux (Ubuntu/Debian):**
    ```bash
    sudo apt update && sudo apt install docker.io docker-compose -y
    sudo usermod -aG docker $USER
    # После этого нужно выйти из системы и зайти снова
    ```

### Шаг 2. Установка Python и создание виртуального окружения
Ansible написан на Python. Чтобы избежать конфликтов версий библиотек, мы создадим изолированное окружение прямо в папке проекта.

> ⚠️ **Важно:** Выполняйте все команды ниже в терминале, находясь в папке проекта (`ansible-lab`).

1.  **Создайте виртуальное окружение:**
    ```bash
    python3 -m venv .venv
    ```
    *(Если ошибка `command not found`, установите пакет: `sudo apt install python3-venv` или `brew install python`)*

2.  **Активируйте окружение:**
    *   **macOS / Linux:**
        ```bash
        source .venv/bin/activate
        ```
    *   **Windows (PowerShell):**
        ```powershell
        .venv\Scripts\Activate.ps1
        ```
    *   **Windows (CMD):**
        ```cmd
        .venv\Scripts\activate.bat
        ```

    ✅ **Проверка:** Слева в строке терминала должна появиться надпись `(.venv)`.

3.  **Установите зависимости:**
    Внутри активного окружения установите Ansible и необходимые библиотеки:
    ```bash
    pip install --upgrade pip
    pip install ansible passlib
    ```

4.  **Проверьте установку:**
    ```bash
    ansible --version
    ```
    Вы должны увидеть версию Ansible без ошибок.

## 🚀 Часть 2. Запуск инфраструктуры (Серверы)

Мы используем Docker Compose, чтобы поднять 3 виртуальных сервера Ubuntu с настроенным SSH и веб-терминалом.

1.  **Соберите и запустите контейнеры:**
    ```bash
    docker-compose up -d --build
    ```
    *Первый запуск может занять 1-2 минуты (скачивание образа и установка пакетов). Последующие запуски будут мгновенными.*

2.  **Проверьте статус:**
    ```bash
    docker-compose ps
    ```
    Вы должны увидеть три контейнера со статусом `Up`: `ansible-node-1`, `ansible-node-2`, `ansible-node-3`.

3.  **Доступ к серверам:**
    *   **Через браузер (Веб-терминал):**
        *   Сервер Russian: http://servers.russian.localhost
        *   Сервер France: http://servers.france.localhost
        *   Сервер Chine: http://servers.chine.localhost
        *(Логин: `root`, Пароль: `student123`)*
    *   **Через SSH (для продвинутых):**
        ```bash
        ssh root@127.0.0.1 -p 2221
        ```

## 📝 Часть 3. Выполнение заданий

Все задания выполняются путем создания `.yml` файлов и запуска их через Ansible.

### Проверка связи
Перед началом убедитесь, что Ansible видит все серверы:
```bash
ansible all -i hosts -m ping
```
Вы должны получить ответ `SUCCESS` от всех трех хостов (`web1`, `web2`, `web3`).

### Список заданий:
1.  **`00_print_messages.yml`**: Вывод сообщений.
2.  **`01_update_packages.yml`**: Обновление пакетов и установка нужных утилит (htop, mc, vim, curl).
3.  **`02_user_management.yml`**: Создание пользователя ithub и подключение к ней с помощью ssh
4.  **`03_introduction.yml`**: Создание текстового файла во всех нодах.
5.  **`04_easy_script.yml`**: Придумать какой нибудь простой скрипт.
6.  **`05_hard_script.yml`**: Придумать какой нибудь суперсложный скрипт .
### Пример запуска задания:
```bash
ansible-playbook -i hosts 01_setup.yml
```

## 🧹 Остановка и очистка

Когда работа завершена:

1.  **Остановить серверы:**
    ```bash
    docker-compose down
    ```
    *(Серверы выключатся, но данные сохранятся. При следующем `up` они включатся быстро).*

2.  **Полная очистка (удаление контейнеров и образов):**
    ```bash
    docker-compose down --rmi all --volumes
    ```

3.  **Деактивация виртуального окружения:**
    Просто введите команду:
    ```bash
    deactivate
    ```

## ❓ Частые проблемы (FAQ)

**Ошибка: `No module named 'passlib'`**
*   **Решение:** Вы забыли установить библиотеку для работы с паролями. Активируйте `.venv` и выполните: `pip install passlib`.

**Ошибка: `Jinja2 environment setup ... cannot import name 'environmentfilter'`**
*   **Решение:** Конфликт версий Python. Убедитесь, что вы активировали виртуальное окружение (`.venv`) перед запуском Ansible. Не используйте системный Python.

**Ошибка: `Docker command not found`**
*   **Решение:** Docker Desktop не запущен или не добавлен в PATH. Запустите приложение Docker Desktop вручную.

**Ansible не видит хосты**
*   **Решение:** Убедитесь, что вы передаете файл инвентаря флагом `-i hosts`. Проверьте, что контейнеры работают (`docker-compose ps`).
