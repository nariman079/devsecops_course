# GitBash - Работа с Git и GitHub

## Описание задания

На третьей паре студенты изучают продвинутые аспекты работы с Git и GitHub, включая настройку SSH-ключей, работу с приватными репозиториями, управление настройками репозиториев и автоматизацию процессов. Особое внимание уделяется безопасной аутентификации и эффективной работе с удаленными репозиториями.

## Теоретическая часть

### 1. SSH-ключи для GitHub

**Зачем нужны SSH-ключи:**
- Безопасная аутентификация без ввода пароля
- Защита от перехвата учетных данных
- Удобство работы с несколькими репозиториями

**Генерация SSH-ключа:**
```bash
# Генерация нового SSH-ключа
ssh-keygen -t ed25519 -C "your_email@example.com"

# Или для старых систем
ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
```

**Активация ssh-agent:**
```bash
# Запуск ssh-agent
eval "$(ssh-agent -s)"

# Добавление ключа в ssh-agent
ssh-add ~/.ssh/id_ed25519
```

**Просмотр публичного ключа:**
```bash
cat ~/.ssh/id_ed25519.pub
```

### 2. Работа с GitHub через SSH

**Клонирование репозитория:**
```bash
# Клонирование по SSH
git clone git@github.com:username/repository.git

# Клонирование по HTTPS (альтернатива)
git clone https://github.com/username/repository.git
```

**Настройка удаленного репозитория:**
```bash
# Добавление удаленного репозитория
git remote add origin git@github.com:username/repository.git

# Проверка удаленных репозиториев
git remote -v

# Изменение URL удаленного репозитория
git remote set-url origin git@github.com:username/new-repository.git
```

### 3. Управление репозиториями на GitHub

**Создание локального репозитория и публикация:**
```bash
# Инициализация локального репозитория
git init

# Добавление файлов
git add .
git commit -m "Initial commit"

# Создание удаленного репозитория через GitHub CLI
gh repo create username/repository --private

# Или добавление существующего удаленного репозитория
git remote add origin git@github.com:username/repository.git

# Первая публикация
git branch -M main
git push -u origin main
```

**Управление настройками репозитория:**
```bash
# Просмотр информации о репозитории
git remote show origin

# Проверка статуса репозитория
git status

# Просмотр истории коммитов
git log --oneline
```

## Самостоятельная работа

### Задание 1: Настройка SSH-ключей

**Цель:** Научиться генерировать и настраивать SSH-ключи для безопасной работы с GitHub

**Инструкция:**
1. Сгенерируйте новый SSH-ключ с комментарием (своей почтой)
2. Запустите ssh-agent в текущей сессии
3. Добавьте созданный ключ в ssh-agent
4. Скопируйте публичный ключ в буфер обмена
5. Добавьте SSH-ключ в свой аккаунт GitHub через веб-интерфейс
6. Проверьте подключение к GitHub

**Проверка:**
```bash
# Проверка подключения к GitHub
ssh -T git@github.com
```

**Ожидаемый результат:** Сообщение об успешной аутентификации

---

### Задание 2: Работа с приватными репозиториями

**Цель:** Научиться создавать, клонировать и управлять приватными репозиториями

**Инструкция:**
1. Создайте новый приватный репозиторий на GitHub через веб-интерфейс
2. Склонируйте созданный репозиторий на локальный компьютер
3. Создайте в локальной копии файлы:
   - `README.md` - описание проекта
   - `src/main.py` - простой Python скрипт
   - `.gitignore` - файл исключений
4. Добавьте файлы в индекс и сделайте коммит
5. Запушьте изменения в удаленный репозиторий
6. Проверьте, что изменения появились на GitHub

**Проверка:**
- Убедитесь, что репозиторий остается приватным
- Проверьте историю коммитов через веб-интерфейс

---

### Задание 3: Создание локального репозитория и публикация

**Цель:** Научиться инициализировать локальный репозиторий и публиковать его на GitHub

**Инструкция:**
1. Создайте новую директорию для проекта
2. Инициализируйте в ней Git-репозиторий
3. Создайте несколько файлов:
   - `index.html` - простая HTML страница
   - `style.css` - файл стилей
   - `script.js` - JavaScript файл
   - `.gitignore` - исключите временные файлы
4. Сделайте первый коммит
5. Создайте новый репозиторий на GitHub через GitHub CLI
6. Свяжите локальный репозиторий с удаленным
7. Опубликуйте код на GitHub

**Проверка:**
- Убедитесь, что все файлы появились в удаленном репозитории
- Проверьте ветку по умолчанию (должна быть `main`)

---

### Задание 4: Управление настройками репозитория

**Цель:** Научиться управлять настройками репозитория, изменять его параметры

**Инструкция:**
1. Создайте новый репозиторий на GitHub
2. Склонируйте его локально
3. Выполните следующие операции:
   - Измените URL удаленного репозитория (если нужно)
   - Добавьте новую ветку `develop`
   - Сделайте коммит в новую ветку
   - Запушьте ветку на GitHub
4. Через веб-интерфейс GitHub:
   - Смените ветку по умолчанию на `develop`
   - Добавьте файл `LICENSE`

5. Локально:
   - Получите обновления с GitHub
   - Проверьте изменения

**Проверка:**
- Убедитесь, что все изменения синхронизированы
- Проверьте настройки репозитория через веб-интерфейс

---

### Задание 5: Преобразование репозитория (публичный ↔ приватный)

**Цель:** Научиться управлять видимостью репозитория

**Инструкция:**
1. Создайте публичный репозиторий на GitHub
2. Склонируйте его и добавьте несколько коммитов
3. Преобразуйте репозиторий в приватный через веб-интерфейс
4. Проверьте, что доступ к репозиторию работает
5. Создайте еще несколько коммитов и запушьте их
6. Преобразуйте репозиторий обратно в публичный
7. Проверьте доступность репозитория для других пользователей

**Проверка:**
- Убедитесь, что все коммиты сохранены при смене видимости
- Проверьте, что SSH-аутентификация продолжает работать

---

### Задание 6: Удаление репозитория

**Цель:** Научиться безопасно удалять репозитории

**Инструкция:**
1. Создайте тестовый репозиторий
2. Добавьте в него несколько файлов и коммитов
3. Создайте локальную копию репозитория
4. Через веб-интерфейс GitHub:
   - Перейдите в настройки репозитория
   - Найдите раздел "Danger Zone"
   - Удалите репозиторий
5. Проверьте, что репозиторий удален с GitHub
6. Локально:
   - Проверьте, что локальная копия осталась
   - Попробуйте сделать коммит (должен быть error)
   - Удалите локальную копию

**Проверка:**
- Убедитесь, что репозиторий удален с GitHub
- Проверьте, что локальная копия может существовать независимо

## Комплексное задание

### Задание 7: Полный цикл работы с репозиторием

**Цель:** Комплексное применение всех изученных навыков

**Инструкция:**
1. Сгенерируйте новый SSH-ключ (если еще не сделали)
2. Добавьте ключ в ssh-agent и на GitHub
3. Создайте новый приватный репозиторий через GitHub CLI
4. Инициализируйте локальный репозиторий с несколькими файлами:

   - `tests/` - тесты

6. Сделайте несколько коммитов в разных ветках
7. Преобразуйте репозиторий в публичный
8. Создайте issue, pull request и milestone через веб-интерфейс
9. Получите все изменения локально

**Проверка:**
- Все ветки должны быть синхронизированы
- Все настройки должны работать корректно
- SSH-аутентификация должна быть активна

## Советы по выполнению

1. **Резервное копирование:** Перед удалением репозитория всегда делайте резервную копию
2. **SSH-ключи:** Используйте современные алгоритмы (ed25519) для лучшей безопасности
3. **GitHub CLI:** Освойте команды GitHub CLI для более эффективной работы
4. **Документация:** Всегда создавайте README.md и .gitignore
5. **Ветки:** Используйте ветки для изоляции изменений
6. **Коммиты:** Делайте осмысленные коммиты с понятными сообщениями

## Полезные команды GitHub CLI

```bash
# Авторизация в GitHub
gh auth login

# Создание репозитория
gh repo create имя_репозитория --private

# Просмотр issues
gh api repos/foo/bar/issues

# Работа с pull requests
gh api repos/foo/bar/pulls

# Просмотр релизов
gh api repos/foo/bar/releases
```
