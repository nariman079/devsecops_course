# GitBash - Базовые команды для DevSecOps

## Описание задания

На первых 2 парах студенты осваивают базовые команды командной строки, которые будут использоваться на протяжении всего курса. Эти команды являются фундаментом для эффективной работы в терминале.

## Базовые команды

### 1. mkdir - Создание директорий

**Назначение:** Создание новых папок (директорий)

**Синтаксис:**
```bash
mkdir имя_директории
```

**Примеры:**
```bash
# Создание одной директории
mkdir my_project

# Создание нескольких директорий
mkdir dir1 dir2 dir3

# Создание вложенных директорий
mkdir -p parent/child/grandchild
```

**Зачем нужно:** Для организации файловой структуры проектов, создания рабочих директорий

---

### 2. cd - Смена директории

**Назначение:** Перемещение по файловой системе

**Синтаксис:**
```bash
cd путь_к_директории
```

**Примеры:**
```bash
# Переход в домашнюю директорию
cd ~

# Переход в конкретную директорию
cd /home/user/projects

# Переход в родительскую директорию
cd ..

# Переход в предыдущую директорию
cd -
```

**Зачем нужно:** Для навигации по файловой системе, перехода между проектами и рабочими директориями

---

### 3. nano - Текстовый редактор

**Назначение:** Редактирование текстовых файлов прямо в терминале

**Синтаксис:**
```bash
nano имя_файла
```

**Горячие клавиши:**
- `Ctrl+O` - сохранить файл
- `Ctrl+X` - выйти из редактора
- `Ctrl+W` - поиск текста
- `Ctrl+K` - вырезать строку
- `Ctrl+U` - вставить строку

**Примеры:**
```bash
# Создание и редактирование нового файла
nano README.md

# Редактирование существующего файла
nano .gitignore
```

**Зачем нужно:** Для быстрого редактирования конфигурационных файлов, скриптов, документации

---

### 4. echo - Вывод текста

**Назначение:** Вывод текста или переменных в терминал

**Синтаксис:**
```bash
echo "текст"
```

**Примеры:**
```bash
# Вывод простого текста
echo "Hello, World!"

# Вывод значения переменной
echo $PATH

# Запись текста в файл
echo "Содержимое файла" > file.txt

# Добавление текста в файл
echo "Новая строка" >> file.txt
```

**Зачем нужно:** Для вывода информации, создания файлов, отладки скриптов

---

### 5. ls - Просмотр содержимого директории

**Назначение:** Вывод списка файлов и директорий

**Синтаксис:**
```bash
ls [параметры] [путь]
```

**Параметры:**
- `-l` - подробный список
- `-a` - показать скрытые файлы
- `-h` - человекочитаемый формат размеров

**Примеры:**
```bash
# Просмотр текущей директории
ls

# Просмотр с подробной информацией
ls -l

# Просмотр скрытых файлов
ls -a

# Просмотр другой директории
ls /home/user
```

**Зачем нужно:** Для навигации по файловой системе, проверки наличия файлов, анализа структуры проектов

---

### 6. cat - Просмотр содержимого файлов

**Назначение:** Вывод содержимого файлов на экран

**Синтаксис:**
```bash
cat имя_файла
```

**Примеры:**
```bash
# Просмотр содержимого файла
cat README.md

# Объединение нескольких файлов
cat file1.txt file2.txt > combined.txt

# Создание нового файла
cat > new_file.txt << EOF
Содержимое файла
EOF
```

**Зачем нужно:** Для просмотра файлов, объединения файлов, создания файлов

---

### 7. ssh-keygen - Генерация SSH ключей

**Назначение:** Создание пары SSH ключей для безопасного подключения

**Синтаксис:**
```bash
ssh-keygen [параметры]
```

**Параметры:**
- `-t rsa` - тип ключа (RSA)
- `-b 4096` - длина ключа
- `-C "комментарий"` - комментарий к ключу

**Примеры:**
```bash
# Генерация RSA ключа
ssh-keygen -t rsa -b 4096 -C "user@example.com"

# Генерация ключа с указанием пути
ssh-keygen -t rsa -b 4096 -f ~/.ssh/my_key
```

**Зачем нужно:** Для безопасного подключения к удаленным серверам, аутентификации на GitHub

---

### 8. eval - Выполнение команд

**Назначение:** Выполнение команд, полученных из других команд

**Синтаксис:**
```bash
eval команда
```

**Примеры:**
```bash
# Запуск команды из переменной
command="echo Hello"
eval $command

# Использование с ssh-agent
eval $(ssh-agent -s)
```

**Зачем нужно:** Для динамического выполнения команд, работы с ssh-agent

---

### 9. ssh-agent - Управление SSH ключами

**Назначение:** Хранение приватных SSH ключей в памяти

**Синтаксис:**
```bash
ssh-agent [команда]
```

**Примеры:**
```bash
# Запуск ssh-agent
eval $(ssh-agent -s)

# Добавление ключа в агент
ssh-add ~/.ssh/id_rsa

# Просмотр добавленных ключей
ssh-add -l
```

**Зачем нужно:** Для удобного управления SSH ключами, избегания многократного ввода паролей

## Практические задания

1. Создайте директорию `devsecops_practice`
2. Перейдите в созданную директорию
3. Создайте файл `notes.txt` с помощью `echo`
4. Откройте файл в `nano` и добавьте несколько строк
5. Просмотрите содержимое директории с помощью `ls -la`
6. Сгенерируйте SSH ключ для GitHub
7. Запустите ssh-agent и добавьте ключ

## Советы

- Используйте `Tab` для автодополнения путей и команд
- Используйте стрелки вверх/вниз для просмотра истории команд
- Комбинируйте команды с помощью `|` (pipe) для перенаправления вывода
- Используйте `man команда` для просмотра справки по команде
