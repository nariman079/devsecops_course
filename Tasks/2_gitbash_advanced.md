## Дополнительные команды

### 1. grep - Поиск текста в файлах

**Назначение:** Поиск текстовых шаблонов в файлах и вывод совпадений

**Синтаксис:**
```bash
grep [параметры] шаблон [файл/директория]
```

**Параметры:**
- `-r` - рекурсивный поиск по директории
- `-i` - игнорировать регистр
- `-n` - показывать номера строк
- `-v` - инвертировать поиск (показывать несовпадения)
- `-E` - использовать расширенные регулярные выражения

**Примеры:**
```bash
# Поиск текста в файле
grep "import" Main.java

# Поиск по всем файлам в директории
grep -r "TODO" .

# Поиск без учета регистра
grep -i "error" log.txt

# Поиск с номерами строк
grep -n "function" script.js

# Рекурсивный поиск Java файлов
grep -r "public class" src/ --include="*.java"
```

**Зачем нужно:** Для поиска кода, логов, конфигураций, анализа больших объемов текста

### 4. find - Поиск файлов

**Назначение:** Поиск файлов и директорий по различным критериям

**Синтаксис:**
```bash
find [директория] [параметры] [критерии]
```

**Параметры:**
- `-name` - поиск по имени файла
- `-type` - тип объекта (f - файл, d - директория)
- `-size` - размер файла
- `-mtime` - время модификации
- `-exec` - выполнение команды для найденных файлов

**Примеры:**
```bash
# Поиск файлов по имени
find . -name "*.txt"

# Поиск только директорий
find . -type d

# Поиск файлов определенного размера
find . -size +10M

# Поиск файлов, измененных за последние 7 дней
find . -mtime -7

# Поиск и удаление временных файлов
find . -name "*.tmp" -exec rm {} \;
```

**Зачем нужно:** Для поиска файлов по сложным критериям, автоматизации рутинных операций, анализа файловой системы

## Самостоятельная работа

### Задание 1: Создание структуры веб-проекта

**Цель:** Практика использования команд mkdir, touch, cp, mv для создания и управления файловой структурой

**Инструкция:**
1. Создайте директорию `my_website` в на рабочем столе
2. Внутри создайте следующую структуру:
   ```
   my_website/
   ├── css/
   │   ├── style.css
   │   └── responsive.css
   ├── js/
   │   ├── main.js
   │   └── utils.js
   ├── img/
   │   ├── logo.png
   │   └── background.jpg
   ├── index.html
   └── about.html
   ```
3. Создайте копию директории `my_website` с названием `my_website_backup`
4. Переименуйте файл `about.html` в `about_us.html`

**Проверка:**
- Используйте `ls -la` для проверки структуры
- Используйте `tree` (если доступен) или `find .` для просмотра всей структуры

### Задание 2: Работа с текстовыми файлами

**Цель:** Практика использования grep

**Инструкция:**
1. Создайте файл `sample.txt` с несколькими строками текста (не менее 20 строк)
2. Добавьте в файл строки с ключевыми словами: "TODO", "ERROR", "WARNING", "INFO"
3. Выполните следующие операции:
   - Найдите все строки с "ERROR" и выведите их с номерами строк
   - Замените все "TODO" на "DONE" в файле

**Проверка:**
- Используйте `cat sample.txt` для просмотра результата
- Проверьте, что все операции выполнены корректно

### Задание 3: Автоматизация рутинных задач

**Цель:** Практика создания простых bash-скриптов для автоматизации

**Инструкция:**
1. Создайте директорию `scripts` в домашней папке
2. Создайте bash-скрипт `backup_files.sh`, который:
   - Создает резервную копию всех .txt файлов в директории `documents`
   - Сохраняет копии в директорию `backup_YYYY-MM-DD` (где YYYY-MM-DD - текущая дата)
   - Выводит количество скопированных файлов
3. Сделайте скрипт исполняемым
4. Запустите скрипт и проверьте результат

**Проверка:**
- Убедитесь, что скрипт работает без ошибок
- Проверьте, что резервные копии созданы правильно
