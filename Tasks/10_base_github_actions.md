# Практическая работа: Понимание основных разделов GitHub Actions

## Цель работы
Понять и запомнить основные разделы и ключевые слова в GitHub Actions workflow.

## Задание
Создать простой workflow файл и разобрать каждый элемент.

## Шаг 1: Создание простого workflow

### [ ] 1.1 Создайте файл workflow
- [ ] В репозитории создайте папку `.github/workflows/`
- [ ] Создайте файл `simple-test.yml`

### [ ] 1.2 Напишите простой workflow
```yaml
name: Simple Test

on: [push]

jobs:
  test-job:
    runs-on: ubuntu-latest
    
    steps:
    - name: Hello Step
      run: echo "Hello World"
      
    - name: Use Action
      uses: actions/checkout@v3
```

## Шаг 2: Разбор каждого раздела

### [ ] 2.1 Раздел `name`
```yaml
name: Simple Test
```
**Что это:** Название вашего workflow  
**Зачем нужно:** Чтобы вы могли видеть название в разделе Actions  
**Пример:** `name: My CI Pipeline`

**Проверка понимания:**
- [ ] Измените название на что-то свое
- [ ] Запушите изменения
- [ ] Проверьте, что в Actions видно новое название

### [ ] 2.2 Раздел `on`
```yaml
on: [push]
```
**Что это:** События, которые запускают workflow  
**Зачем нужно:** Чтобы workflow запускался автоматически  
**Примеры:**
- `on: [push]` - при любом push
- `on: [pull_request]` - при pull request
- `on: [push, pull_request]` - при push и pull request

**Проверка понимания:**
- [ ] Измените на `on: [pull_request]`
- [ ] Создайте pull request
- [ ] Убедитесь, что workflow запустился

### [ ] 2.3 Раздел `jobs`
```yaml
jobs:
  test-job:
```
**Что это:** Блок с заданиями (работами)  
**Зачем нужно:** Чтобы определить, что будет делать workflow  
**Пример:** Вы можете иметь несколько jobs: `build`, `test`, `deploy`

**Проверка понимания:**
- [ ] Добавьте второй job с другим именем
- [ ] Убедитесь, что оба job работают

### [ ] 2.4 Раздел `runs-on`
```yaml
runs-on: ubuntu-latest
```
**Что это:** На какой операционной системе запускать job  
**Зачем нужно:** Чтобы указать среду выполнения  
**Примеры:**
- `runs-on: ubuntu-latest` - Linux
- `runs-on: windows-latest` - Windows
- `runs-on: macos-latest` - macOS

**Проверка понимания:**
- [ ] Попробуйте изменить на `windows-latest`
- [ ] Проверьте, запускается ли workflow

### [ ] 2.5 Раздел `steps`
```yaml
steps:
```
**Что это:** Список шагов, которые нужно выполнить  
**Зачем нужно:** Чтобы определить последовательность действий  
**Пример:** Каждый шаг - это одна команда или действие

**Проверка понимания:**
- [ ] Добавьте еще один шаг
- [ ] Убедитесь, что все шаги выполняются по порядку

### [ ] 2.6 Раздел `run`
```yaml
- name: Hello Step
  run: echo "Hello World"
```
**Что это:** Команда, которую нужно выполнить  
**Зачем нужно:** Чтобы запустить bash команду  
**Примеры:**
- `run: echo "Hello"` - вывести текст
- `run: ls` - показать файлы
- `run: npm install` - установить зависимости

**Проверка понимания:**
- [ ] Измените команду на `run: ls`
- [ ] Проверьте, что в логах виден список файлов

### [ ] 2.7 Раздел `uses`
```yaml
- name: Use Action
  uses: actions/checkout@v3
```
**Что это:** Использование готового действия из GitHub Marketplace  
**Зачем нужно:** Чтобы не писать команды самому  
**Примеры:**
- `uses: actions/checkout@v3` - скопировать код
- `uses: actions/setup-node@v3` - установить Node.js

**Проверка понимания:**
- [ ] Добавьте `uses: actions/setup-node@v3`
- [ ] Проверьте, что Node.js установился
