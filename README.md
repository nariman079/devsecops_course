# Развёртывание  веб-приложения на Spring Boot от git push до production

## Описание

Этот курс посвящен изучению DevSecOps практик, включая работу с Git, CI/CD, Docker и Docker Compose. Курс рассчитан на 32 пары и охватывает все основные аспекты современной DevOps-инфраструктуры.

## Структура Курса

### 1. Git и GitHub (8 пар)

#### Цели:
- Научиться создавать pull request
- Добавлять комментарии к коду
- Работать с интерфейсом pull request (отклонять, принимать)

#### Практические задания:
- Создание и управление ветками
- Работа с pull request
- Комментирование кода
- Ревью изменений

#### Зачем это нужно:
Эти навыки необходимы для эффективной командной разработки. Pull request позволяют проводить код-ревью, предотвращать ошибки и поддерживать качество кода. Комментирование кода помогает в обсуждении решений и улучшении архитектуры.

### 2. CI/CD Pipeline (8 пар)

#### Цели:
- Создание папки `.github/workflows/`
- Написание файлов: `main.yml`, `ci.yml`, `dev.yml`, `prod.yml`
- Понимание структуры pipeline

#### Практические задания:
- Создание базового pipeline
- Написание CI pipeline с 3-4 этапами
- Работа с разделами: jobs, steps, run
- Выполнение bash операций в pipeline
- Создание телеграм-бота на Java с CI pipeline

#### Телеграм-бот проект:
- Запуск бота в фоновом режиме
- Рассылка сообщений пользователям
- Управление процессами в pipeline
- Работа с интерфейсом GitHub Actions
- Использование переменных и окружений

#### Зачем это нужно:
CI/CD пайплайны автоматизируют процесс развертывания, тестирования и доставки кода. Это позволяет быстро и безопасно выпускать обновления, минимизировать человеческий фактор и обеспечивать стабильность приложений.

### 3. Повторение и Тестирование (1 пара)

#### Цели:
- Закрепление пройденного материала
- Практическое тестирование навыков

#### Зачем это нужно:
Повторение помогает закрепить полученные знания и выявить пробелы в понимании материала.

### 4. Docker (4 пары)

#### Цели:
- Освоение базовых команд Docker
- Создание Dockerfile
- Работа с volumes, networks, environments
- Автоматизация создания Dockerfile
- Установка и настройка Ubuntu в качестве виртуального сервера с SSH доступом

#### Практические задания:
- Базовые команды: start, run, build, ps, ps -a, stop
- Создание Dockerfile с командами echo
- Загрузка файлов в контейнер (COPY)
- Запуск Java проектов в контейнерах
- Создание volumes, networks, environments
- Подключение к контейнеру через SSH
- Использование команды docker exec -it

#### Зачем это нужно:
Docker позволяет изолировать приложения в контейнерах, что обеспечивает переносимость, масштабируемость и упрощает развертывание. Это основа современных облачных технологий.

### 5. Docker Compose (5 пар)

#### Цели:
- Освоение команд docker compose
- Написание docker-compose.yml файлов
- Работа с несколькими сервисами

#### Практические задания:
- Команды: up, up --build, down, down -v
- Разделы: build, image, command, volumes, networks, services, healthcheck
- Создание compose с Java проектом
- Интеграция Redis + PostgreSQL + Java
- Телеграм-бот в docker-compose.yml

#### Зачем это нужно:
Docker Compose позволяет управлять несколькими контейнерами как единым приложением. Это необходимо для создания сложных систем с несколькими взаимодействующими сервисами.

### 6. Docker Hub (2 пары)

#### Цели:
- Работа с Docker Hub
- Публикация образов

#### Практические задания:
- Регистрация и создание токена
- Настройка доступа через терминал
- Билд и пуш образов
- Проверка публикации

#### Зачем это нужно:
Docker Hub - это центральный реестр образов, где можно хранить и делиться контейнерами. Это важно для командной разработки и распределения приложений.

### 7. Интеграция Docker + CI/CD + Docker Hub (3 пары)

#### Цели:
- Комплексное использование всех технологий
- Автоматизация процессов

#### Практические задания:
- Создание репозитория с CI pipeline
- Использование docker buildx
- Запуск образов из Docker Hub
- Настройка сервера с Docker и SSH

#### Зачем это нужно:
Интеграция всех технологий позволяет создавать полностью автоматизированные процессы развертывания, от кода до production окружения.

## Итог

Курс охватывает 44 пары и дает комплексные знания по DevSecOps практикам. По окончании курса вы сможете:

- Эффективно работать в команде с использованием Git
- Создавать и настраивать CI/CD пайплайны
- Контейнеризировать приложения с помощью Docker
- Управлять многокомпонентными системами с помощью Docker Compose
- Автоматизировать процессы развертывания
- Работать с облачными технологиями

Эти навыки востребованы на рынке труда и позволяют эффективно разрабатывать, тестировать и развертывать современные приложения.
